`n SEMESTER PROJECT SPECIFICATION `n EduLearn `n Online Learning Platform `n Read Lectures. Attempt Quizzes. Earn Certificates. `n Stack `n MongoDB, Express, React, Node.js `n Domain `n E-Learning / EdTech `n Features `n 23 Required Features Covered `n `n `n `n 1. Executive Summary `n EduLearn is a full-stack online learning platform built on the MERN stack (MongoDB, Express.js, React, Node.js). It enables instructors to publish structured courses composed of lecture modules, and allows students to study at their own pace, attempt auto-graded quizzes, and earn downloadable completion certificates. The platform addresses all three pillars of the course framework: fundamental business logic (course and quiz management), personal productivity (progress tracking, personalization, and PDF certificates), and collaboration (notifications, blog discussions, and real-time updates). `n `n 2. Project Overview `n 2.1 Vision `n To provide an accessible, secure, and engaging online learning experience that empowers students to master subjects through structured lectures and formative quizzes, while equipping instructors with powerful tools to create and monetize educational content. `n 2.2 Core User Roles `n Student — browse courses, read lectures, attempt quizzes, track progress, earn certificates, write blog comments. `n Instructor — create and manage courses, upload lecture content, author quiz questions, view earnings analytics. `n Admin — manage all users and content, view platform-wide analytics, configure platform settings. `n 2.3 Key Workflows `n Student Learning Flow `n Register / log in → Browse & search courses `n Enrol in a course (free or paid via Stripe) `n Read lecture modules in sequence `n Attempt end-of-module quizzes; receive instant feedback `n Complete all modules → Generate and download PDF certificate `n Instructor Content Flow `n Create a course with title, description, category, and price `n Add lecture modules (rich text + optional video embed) `n Author multiple-choice / true-false quiz questions per module `n Publish course → students can enrol `n Monitor earnings in the cash flow dashboard `n `n 3. Technical Architecture `n 3.1 MERN Stack Breakdown `n Layer `n Technology `n Role `n Frontend `n React 18 + Redux Toolkit `n SPA user interface, state management, routing (React Router v6) `n Backend `n Node.js + Express.js `n REST API, middleware, authentication, business logic `n Database `n MongoDB + Mongoose `n Document storage for users, courses, quizzes, transactions, notifications `n Real-Time `n Socket.io `n Live notifications and quiz submission feedback `n Media `n Cloudinary `n Image and video storage for lecture thumbnails and avatars `n Payments `n Stripe API `n Secure course purchase and instructor payout processing `n Email `n Nodemailer (Gmail SMTP) `n Transactional emails: verification, password reset, certificates `n PDF Gen `n Puppeteer / pdfkit `n Server-side PDF certificate generation `n 3.2 Folder Structure `n edulearn/ ├── client/ (React + Redux) ├── server/ (Express + Mongoose) └── shared/ (types, constants) `n client/src/components — Reusable UI components (Navbar, Sidebar, QuizCard, LectureReader, etc.) `n client/src/pages — Route-level page components (Dashboard, CoursePage, QuizPage, AdminPanel, etc.) `n client/src/store — Redux slices (authSlice, courseSlice, quizSlice, notificationSlice) `n server/routes — Express routers grouped by resource `n server/models — Mongoose schemas (User, Course, Lecture, Quiz, Question, Attempt, Transaction, Notification, Blog) `n server/middleware — Auth guard, role check, error handler, rate limiter `n `n 4. Core Data Models `n All models are implemented with Mongoose. Key relationships are summarised below. `n Model `n Key Fields `n Notes `n User `n name, email, passwordHash, role, avatar, bio, enrolledCourses[], preferences `n Role enum: student | instructor | admin `n Course `n title, description, instructor (ref), modules[], price, category, rating, isPublished `n Embedded module list for ordering `n Lecture `n course (ref), module, title, content (HTML), videoUrl, order, isPreview `n Rich HTML stored as string `n Quiz `n lecture (ref), questions[], passMark, timeLimit `n One quiz per lecture `n Question `n quiz (ref), text, options[], correctIndex, explanation `n MCQ and true/false support `n Attempt `n student (ref), quiz (ref), answers[], score, passed, completedAt `n Stores full answer history `n Transaction `n user (ref), course (ref), type (debit/credit), amount, status, createdAt `n Supports cash flow statements `n Notification `n recipient (ref), type, message, isRead, link, createdAt `n Used by Socket.io + bell icon `n Blog `n author (ref), title, content, tags[], comments[], publishedAt `n Instructor and admin authored `n `n 5. Feature Deep-Dives `n 6.1 Dashboard, Navbar & Sidebar (Req. 1) `n The dashboard is the first screen a logged-in user sees. It is divided into three zones: `n Top Navbar — Logo, global search bar, notification bell (live badge count), and user avatar dropdown (Profile, Settings, Logout). `n Left Sidebar — Collapsible panel listing: My Courses, Browse, Bookmarks, My Certificates, Earnings (instructors only), and Admin Panel (admins only). Active route is highlighted. `n Main Content Area — Responsive grid of cards: In-Progress Courses (with % completion bars), Recommended Courses, Recent Quiz Scores, and Upcoming Deadlines. `n Redux manages sidebar open/close state globally so the layout persists across navigations. `n 6.2 Authentication & Authorization (Req. 2 & 10) `n EduLearn implements layered security: `n Registration — Email + password (hashed with bcrypt, salt rounds = 12). Email verification link sent via Nodemailer. `n Login — Returns signed JWT (1 hour expiry) + refresh token (7 days, stored in HttpOnly cookie). `n Protected Routes — Express middleware verifies JWT on every private API call. React Router guards redirect unauthenticated users to /login. `n Role-Based Access Control — Three roles (student, instructor, admin). Middleware checks role before granting access to sensitive endpoints. `n Rate Limiting — express-rate-limit caps /auth/* at 10 requests per 15 minutes to prevent brute-force attacks. `n Helmet.js — Sets secure HTTP headers (CSP, HSTS, X-Frame-Options, etc.). `n 6.3 Lecture Reader & Quiz Engine (Req. 8, 9, 13) `n The Lecture Reader is a focused reading interface: `n Left panel: module/lecture index (sidebar collapses on mobile). `n Centre panel: lecture content rendered from stored HTML (React-Quill output). `n Right panel (desktop): related notes and instructor tips. `n &apos;Mark as Complete&apos; button persists progress to the backend and unlocks the quiz. `n The Quiz Engine presents questions one at a time with a countdown timer. On submission, the server grades answers against the stored correct indices, returns the score, and — if passed — triggers certificate generation and a notification. `n 6.4 AI Recommendation System (Req. 12) `n The recommendation engine runs server-side in a dedicated Express service: `n Content-Based Filtering — Courses are tagged with subject, difficulty, and skill keywords. When a student enrols, the system finds courses with overlapping tags. `n Collaborative Filtering — The system identifies students with similar enrolment histories and surfaces courses they completed with high scores. `n Score Weighting — Quiz pass rate and completion rate for each course influence the ranking of recommendations. `n API Endpoint — GET /api/recommendations/:userId returns a ranked list of up to 10 course IDs, cached in MongoDB for 24 hours. `n 6.5 PDF Certificate Generation (Req. 15) `n When a student passes all quizzes in a course: `n The server triggers a Puppeteer headless browser to render an HTML certificate template with the student name, course title, completion date, and a unique UUID verification code. `n The rendered PDF is uploaded to Cloudinary and the URL is stored in the Attempt record. `n A notification is dispatched to the student, and the certificate appears in their profile under My Certificates. `n A public verification page (/verify/:uuid) allows anyone to confirm certificate authenticity without logging in. `n 6.6 Cash Flow Statements (Req. 14) `n Instructors see a financial dashboard with: `n Credit entries — Each new enrolment in their course creates a credit Transaction record with the net amount (after platform fee). `n Debit entries — Stripe payout fees and refunds are recorded as debit transactions. `n Ledger View — A paginated table of all transactions, filterable by date range and type. `n Summary Cards — Total earned, total paid out, pending balance. `n Revenue Chart — Line chart (Chart.js) showing monthly earnings over the past 12 months. `n 6.7 Analytics Dashboard (Req. 21) `n The admin analytics panel surfaces platform health metrics: `n Daily Active Users (DAU) — Aggregated from login timestamps. `n New Registrations — Bar chart by day/week/month. `n Top Courses by Enrolment — Ranked table. `n Quiz Pass Rates — Per-course heatmap. `n Revenue Overview — Platform total vs instructor payouts. `n All charts use Recharts components driven by MongoDB aggregation pipeline results returned via dedicated /admin/analytics/* Express routes. `n 6.8 Real-Time Notifications (Req. 22) `n Notification delivery uses a hybrid push + pull approach: `n Socket.io — On login, the React client opens a socket connection authenticated by JWT. The server emits events to the user&apos;s room when relevant actions occur. `n MongoDB Persistence — Every notification is also saved to the Notification collection so users can retrieve missed alerts after reconnection. `n Redux Slice — The notificationSlice stores the notification list and unread count; the bell icon badge re-renders reactively. `n Types — new_quiz, grade_released, comment_reply, certificate_ready, new_enrolment (instructors), admin_broadcast. `n `n 7.
